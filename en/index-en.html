Navigation Structure

Our navigation structure is based on the menu on the left side, where each option defines what type of technology will be used in the origin request. Service registration would be REST origin, database registration would be database origin, SOAP registration would be SOAP origin, file registration would be FTP/SFTP origin for reading any type of file (TXT, CSV, XML...), SAP-RFC registration would be for specific RFC's or BAPI's from the SAP world.

RUNNING SERVICES:

In the Control Panel menu in the Running Services option, you will have access to all active queues and each connector on our platform has its own queue. That is, there will be a queue for each theme with the option to "STOP" the queue, "RESTART", or even "DELETE" messages.

VALIDATE COMMUNICATION:

Serves as a facilitator for the platform to test communication with the provided URL via TELNET.

DATA EXTRACTOR:

Tool similar to SQL studio. Here it is possible to run syntaxes on your database, if it is your model and export to excel or even validate the syntax of queries.

GLOBAL VARIABLES:

It is a way to register contents, either to not leave it exposed in the connector or even to facilitate constructions using these variables without having to repeat contents.

NOTIFICATION CENTER:

Creates alerts actively, that is, the platform sends an email according to what was configured. Example: You want to create an alert so that whenever the destination response returns ERROR 500, an email is automatically sent to IT.

FILE UPLOAD:

Upload csv, txt files to an N4LINK table, with the possibility of accessing the contents through connectors. Example: To generate e-commerce sales for the ERP, specific registration is mandatory, but the data is not transmitted from the e-commerce, but with this function, it is possible to create a rule from an e-commerce field to an Excel spreadsheet and determine the correct content to send to the ERP.

Running Services

Here as previously reported, these are the active queues on our platform, where the connector ID is shown after the hyphen in the QUEUE column:

In this option, it is possible to check the active queues on the N4LINK platform, where the connector ID is shown after the hyphen in the QUEUE column.

QUEUE COLUMN:
Your company code on the N4LINK platform + connector ID is displayed.

COMPANY COLUMN:
The registered company name is displayed.

STATUS COLUMN:
Indicates the current status of the queue:

  • GREEN indicates it is running;
  • YELLOW indicates it is in the process of stopping/restarting;
  • RED indicates it is stopped.

ACTIONS COLUMN:

  • DELETE BUTTON - eliminates queue messages, then stops the queue and brings it up again
  • PURGE BUTTON - eliminates current queue messages
  • Red button (-) is the button that stops the queue
  • RESTART button - blue with two arrows facing each other - serves to stop and bring up the queue again
  • PLAY button - green - starts the queue that is stopped

Parent Connector Concept

There is the possibility of chaining connectors for data enrichment or even for logical sequence reasons. Example: Your company is integrating orders from an ecommerce. Mandatorily, before downloading the order it is necessary to register the customer. Here, it could be configured for the order connector to call the customer connector first and only after this process, the customer is integrated to download the sales order. It is possible to inform the parent connector, what is the execution type: if it is executed record by record or only after the last record integrated to the parent connector or, even, if it should continue in case of error in the parent connector or not.
